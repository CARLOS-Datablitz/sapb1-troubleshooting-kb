du - Es el comando principal que significa "disk usage" (uso del disco). Se utiliza para mostrar el espacio que ocupan archivos y directorios.

-s - Opción "summarize". Muestra solo el total del directorio especificado, sin desglosar el contenido de subdirectorios. Sin esta opción, du mostraría el tamaño de cada subdirectorio por separado.

-h - Opción "human-readable". Muestra los tamaños en formato legible para humanos usando unidades como K (kilobytes), M (megabytes), G (gigabytes), etc., en lugar de mostrar solo bytes.

===== Sistemas de particiones Windows ====
MBR y GPT no son sistemas de archivos, sino esquemas de particionado (partition schemes) que definen cómo se organiza y estructura el disco a nivel de particiones.

MBR (Master Boot Record)

Limitaciones: Máximo 4 particiones primarias, discos hasta 2TB
Antiguo: Estándar desde los años 80
Compatibilidad: Compatible con sistemas muy antiguos
Estructura: Tabla de particiones de 64 bytes


GPT (GUID Partition Table)

Moderno: Parte del estándar UEFI
Sin limitaciones prácticas: Hasta 128 particiones, discos de múltiples terabytes
Redundancia: Mantiene copias de respaldo de la tabla de particiones
Requerido: Para discos mayores a 2TB

Cómo se relacionan
Cuando configuras un disco nuevo, el proceso es:

Primero eliges el esquema de particionado (MBR o GPT)
Después creas las particiones
Finalmente formateas cada partición con un sistema de archivos (NTFS, ReFS, etc.)

Para un servidor Windows moderno, definitivamente usa GPT porque:

Soporta discos grandes sin límite práctico
Es más confiable y resistente a errores
Es el estándar actual
